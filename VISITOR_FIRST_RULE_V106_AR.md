# قاعدة التطوير — v10.6

## السؤال الوحيد قبل أي تحسين أو تطوير أو تغيير

```text
هل هذا ينفع الزائر أو صاحب المشروع مباشرة؟
```

## القرار

### نعم
يمكن أن يظهر في الواجهة أو الكود.

### لا
مكانه ملف داخلي أو توثيق، وليس الواجهة.

### غير واضح
لا تتم إضافته حتى يتضح أثره المباشر.

## ما يستحق الظهور في الواجهة

- تصفح الفرص.
- فهم تفاصيل الفرصة.
- حفظ ومشاركة الفرص.
- إرسال اهتمام.
- طلب NDA عند الحاجة.
- إضافة مشروع.
- الحساب ومتابعة الطلبات.
- الثقة والسياسات الضرورية.
- تجربة جوال واضحة.

## ما يبقى داخليًا

- خطط التشغيل والصيانة.
- فرز البلاغات.
- حوكمة التطوير.
- تجارب النمو.
- ملاحظات التحليلات.
- قوالب الدعم.
- أدلة المبيعات.
- خطط الرجوع.

## قاعدة حاسمة

الواجهة ليست مكانًا لخطط الإدارة الداخلية.  
الواجهة مكان لخدمة الزائر وصاحب المشروع فقط.
