# خطة قياس الأحداث — v9.2

## الهدف

معرفة أين يحدث التحول الحقيقي في المنصة:

زيارة → سوق → تفاصيل فرصة → اهتمام / NDA / حفظ / مشاركة.

## الأحداث الأساسية المقترحة

| الحدث | المعنى |
|---|---|
| `view_marketplace` | دخول المستخدم إلى سوق الفرص |
| `view_opportunity` | فتح صفحة تفاصيل فرصة |
| `save_opportunity` | حفظ فرصة |
| `share_opportunity` | مشاركة فرصة |
| `start_interest` | بدء نموذج الاهتمام |
| `submit_interest` | إرسال اهتمام |
| `start_nda` | بدء طلب NDA |
| `submit_nda` | إرسال NDA |
| `start_project_submission` | بدء إضافة مشروع |
| `submit_project` | إرسال مشروع |
| `open_account` | فتح الحساب |

## مؤشرات أسبوعية

- عدد زيارات السوق.
- نسبة فتح تفاصيل الفرصة من السوق.
- عدد طلبات الاهتمام.
- عدد طلبات NDA.
- عدد المشاريع المضافة.
- أكثر صفحة يخرج منها المستخدم.
- أكثر CTA يتم الضغط عليه.

## القرار بناءً على البيانات

- زيارات كثيرة وطلبات قليلة: المشكلة في عرض الفرص أو CTA.
- فتح تفاصيل كثير وطلبات قليلة: المشكلة في تفاصيل الفرصة أو الثقة.
- طلبات كثيرة وغير جادة: المشكلة في نموذج الاهتمام.
- مشاريع قليلة: المشكلة في صفحة إضافة المشروع أو عرض القيمة لصاحب المشروع.

## ملاحظة تنفيذية

تمت إضافة ملف:

```text
lib/analyticsEvents.js
```

وهو لا يرسل البيانات إلى طرف خارجي. هو جاهز للربط لاحقًا مع أداة مثل Google Analytics أو PostHog أو Vercel Analytics عند الحاجة.
