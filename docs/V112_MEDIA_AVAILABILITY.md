# v112 — خطوة "الصور والمستندات" (إفصاح بلا رفع)

تنفيذ الخيار (أ): التقاط توفّر الصور دون أي رفع ملفات أو تخزين،
المناسب لمرحلة التجربة الحالية.

## ما أُضيف

### نموذج إضافة المشروع (components/Forms.jsx)
- خطوة جديدة (الخطوة 3): "الصور والمستندات".
- سؤال: "هل تتوفر صور للمشروع؟" (نعم / سأوفّرها لاحقًا / لا تتوفر حاليًا).
- اختيار متعدد لما هو متاح: صور موقع، صور منتجات، مخطط، مستندات ورخص، فيديو.
- حقل ملاحظة اختياري.
- ملاحظة خصوصية واضحة (تتوافق مع PDPL): لا ترسل بيانات حساسة قبل تنظيم السرية.
- لا رفع ملفات ولا Supabase Storage. الصور تُطلب لاحقًا عند جدّية المهتم.

### إعادة ترقيم الـ wizard
- أصبح 5 خطوات: بيانات المشروع، الأرقام، الصور والمستندات، التواصل، المراجعة.
- تم تحديث: التحقق، حدود next/prev، شروط العرض، الأزرار، الحقول في payload ونص واتساب والمراجعة.

### قاعدة البيانات (supabase/media-availability-v111.sql)
- إضافة أعمدة: has_media, media_types, media_note إلى project_submissions.
- آمن لإعادة التشغيل. لا تغيير في RLS.

## إصلاح عطل ضمني (bug fix مهم)
نسخة v111 كان فيها حقل commission_acknowledged في الحالة، ومستخدم في
التحقق (يمنع التقدّم من الخطوة steps.length - 1)، وفي ملخص المراجعة —
لكن **بلا أي checkbox UI** للمستخدم ليؤكده. هذا كان يجعل النموذج يصل
لطريق مسدود لا يمكن تجاوزه.

الإصلاح: أُضيف checkbox إقرار عمولة النجاح في خطوة المراجعة (الخطوة 5)،
ونُقل التحقق ليبوّب الإرسال النهائي (step === steps.length) بدل منتصف
المعالج. زر الإرسال يبقى معطّلًا حتى تأكيد الإقرار.

2. شغّل npm run build محليًا للتأكد.
