# v117 — تنظيف CSS الميت (بلا أي تغيير بصري)

## الخلفية
تراكمت في globals.css أصناف مرقّمة بالإصدار (-vNN) من ميزات وصفحات
حُذفت سابقًا (المكوّنات التشغيلية، صفحتا compare/discover، صفحات
loading/system القديمة، نسخ marketplace v14/v41 القديمة).

## ما تم
- تحليل 332 صنفًا مرقّمًا في CSS مقابل استخدامها الفعلي في ملفات JSX.
- حذف قواعد الأصناف الميتة فقط (85 صنفًا بلا أي استخدام في JSX).
- قاعدة أمان صارمة: أي قاعدة تشارك المُحدّد (selector) مع صنف حيّ
  لم تُحذف (بقيت 12 حالة مشتركة سليمة).

## النتيجة
- globals.css: من 11,850 سطرًا إلى 6,828 سطرًا (~42% أقل، ~30KB).
- صفر تغيير بصري: كل صنف مستخدم احتفظ بتنسيقه كاملًا (247 صنفًا حيًّا سليم).
- لا أخطاء بنيوية: الأقواس متوازنة، لا قواعد فارغة، لا بقايا فواصل.

## التحقق
- 69 ملف JSX: بنية سليمة، الاستيرادات تطابق التصديرات.
- فحص تكامل CSS: لا {} فارغة، لا @media فارغة، لا selector-list artifacts.

## ملاحظة مهمة للمستقبل (لمنع تكرار المشكلة)
عند أي تحسين بصري قادم، **عدّل الصنف القائم مباشرة** بدل إضافة صنف
-vNN جديد فوقه. النمط الحالي (marketplace-card-v41 + marketplace-card-v116
معًا على نفس العنصر) يراكم كودًا ميتًا ويسبب تعارضات يصعب تتبعها.
القاعدة: صنف واحد لكل غرض، يُعدّل لا يُستنسخ.
