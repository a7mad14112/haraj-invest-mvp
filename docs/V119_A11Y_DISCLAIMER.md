# v119 — إمكانية الوصول + توحيد إخلاء المسؤولية

## 1. إمكانية الوصول (ربط التسميات بالحقول)
كانت تسميات النماذج (<label>) مجرد نص مجاور للحقول، دون ربط برمجي —
فقارئات الشاشة لا تعرف أي تسمية تخص أي حقل (إقصاء لمستخدمي قارئات الشاشة).

الإصلاح (38 حقلًا عبر نموذجين):
- الحقول أحادية السطر: لُفّ <input> داخل <label> (ربط ضمني) مع <span class="lbl"> للنص.
- القوائم والمناطق النصية: رُبطت عبر htmlFor/id بمعرّفات فريدة.
- صفر تغيير بصري (تم التحقق برسم اختبار): الحقول تبدو مطابقة تمامًا.

النتيجة: كل حقل إدخال الآن مرتبط بتسميته، يعمل مع قارئات الشاشة،
والنقر على التسمية يركّز الحقل.

## 2. توحيد إخلاء المسؤولية على صفحات الفرص
عبارة "ليست توصية استثمارية" كانت في صفحات الشروط/الخصوصية/العمولة فقط،
وغائبة عن صفحة تفاصيل الفرصة والسوق — حيث يقيّم الزائر فرصة فعلية.

الإصلاح:
- إضافة إخلاء مسؤولية موحّد وواضح في أسفل صفحة تفاصيل الفرصة.
- إضافة نسخة مختصرة في أسفل صفحة السوق.
- صياغة متّسقة مع صفحة إخلاء المسؤولية الرسمية، مع رابط إليها.

أهمية ذلك: يقوّي موقفك القانوني (متّسق مع إزالة ادعاءات ترخيص CMA
سابقًا) ويوضّح للزائر طبيعة المنصة عند نقطة التقييم الفعلية.

## التحقق
- 68 ملف: بنية سليمة، الاستيرادات تطابق التصديرات، الوسوم متوازنة.
- صفر تغيير بصري على النماذج (تم التحقق بالرسم).
- لا تغيير في Supabase/Auth/Admin. لا SQL جديد.
