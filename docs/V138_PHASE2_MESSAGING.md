# v138 — المرحلة 2: نظام المحادثات الداخلي (المهتم ↔ الأدمن)

## ما أُضيف
1. supabase/phase2-conversations-v138.sql — جدولا conversations و messages + سياسات RLS.
2. lib/conversations.js — منطق المحادثات + دالة حجب الأرقام (maskContactInfo).
3. app/messages/ — صفحة "محادثاتي" (صندوق وارد + محادثة + إرسال).
4. components/Forms.jsx — عند إرسال طلب اهتمام، تُنشأ محادثة موثّقة وتبدأ برسالة المهتم.
5. components/Header.jsx — رابط "محادثاتي" للمستخدم المسجّل.
6. app/admin/AdminPanel.jsx — جدولا "المحادثات" و"الرسائل" للمراجعة.
7. globals.css — تنسيقات المحادثات (v1.38).

## كيف يعمل
- المهتم يرسل طلب اهتمام (بعد تسجيل الدخول) → تُنشأ محادثة + رسالة أولى.
- يتابع من "محادثاتي".
- كل رسالة: تُحجب فيها أرقام التواصل تلقائيًا (body)، ويُحفظ الأصل للأدمن (body_raw) مع علامة was_masked.
- الأدمن يرد عبر Supabase (أو لوحة الإدارة) — ردوده عبر service role تتجاوز RLS.

## حجب الأرقام (مُختبَر)
يحجب: الجوال (0501234567 و ٠٥٠...)، الإيميل، روابط واتساب/تيليجرام/انستغرام.
لا يحجب: الأرقام المشروعة في النص (الأسعار، الأرباح).
ملاحظة: لا يلتقط الحروف ("صفر خمسة") أو الصور — الردع الكامل قانوني/تجاري.

## التفعيل
1. شغّل supabase/phase2-conversations-v138.sql في Supabase.
2. انشر، وامسح كاش الموقع.
3. اختبر: سجّل دخول → أرسل اهتمام على فرصة → افتح "محادثاتي" → جرّب إرسال رسالة فيها رقم (يجب أن يُحجب).

## رد الأدمن (مؤقتًا)
من Supabase → Table Editor → messages → أضف صفًا: conversation_id، sender_role='admin'، body=ردّك.
لاحقًا (مرحلة 5): محرّر رد داخل لوحة الإدارة + إشعارات.
